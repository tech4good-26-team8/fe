# Familog — 프론트엔드 프로젝트 설명

시니어 디지털 격차 해소 / 가족 단위 커뮤니티 · 1-Day 해커톤 (프론트엔드 파트)

> 이 문서는 `familog-server`의 `docs/01_PROJECT_OVERVIEW.md`와 짝을 이루는 프론트 버전이다.

---

## 1. 프론트가 담당하는 범위

Familog는 가족이 올린 소식을 시니어가 **가족 캐릭터를 보고, 가족 목소리로 듣는** 서비스다. 프론트엔드는 이 경험이 실제로 일어나는 **화면 자체**를 만든다.

- 가족(발신자): 가입(사진+목소리 등록) → 소식(사진+글) 올리기
- 시니어(수신자): 홈에서 가족 아바타 보기 → 탭해서 목소리로 듣기

백엔드(familog-server, familog-ai)가 아바타·보이스팩·낭독을 "생성"한다면, 프론트는 그 생성물을 **가장 조작이 적은 방식으로 시니어에게 전달**하는 역할이다.

---

## 2. 왜 이렇게 설계했나

시니어 서비스에서 프론트의 최우선 원칙은 하나다: **조작을 줄이고, 큰 시각·청각 신호로 대체한다.**

- 작은 글씨 대신 큰 원형 아바타 카드
- 텍스트를 읽는 대신 탭 한 번으로 음성 재생
- 복잡한 메뉴 대신 하단 탭 2개(채팅/갤러리) + 중앙 음성 버튼 하나

가족(발신자) 쪽 화면은 상대적으로 일반적인 UX(입력창, 키보드)를 허용하지만, 시니어가 보는 홈·채팅 화면은 고대비 색상과 큰 터치 영역을 우선한다.

---

## 3. 누가 쓰나 (화면 관점)

| 구분 | 화면에서 하는 일 |
|---|---|
| 시니어 | 홈에서 아바타 탭 → 목소리로 듣기. 음성 버튼으로 답장 녹음. |
| 가족 | 가입 시 사진 촬영(아바타용) → 초대코드 공유 → 채팅에서 사진/음성/텍스트 업로드 |

지금 구현은 두 역할을 화면으로 명확히 분리하지 않고 하나의 앱 안에서 같은 컴포넌트를 공유한다(1-Day 해커톤 범위 판단). 시니어 전용 모드가 필요해지면 이후 라우트를 분기할 수 있다.

---

## 4. 핵심 화면 흐름

**온보딩 (가입)**
1. 이름 입력 → 가족 코드로 접속 / 새 가족 만들기
2. 초대코드 입력 (6자리)
3. 카메라 스캔 → 3D 캐릭터 아바타용 사진 촬영
4. 초대 완료 → 홈으로 이동

**홈 (시니어 소비 + 가족 발신 공용)**
- 가족 아바타 격자, 최근 소식 실시간 피드(목데이터)
- 중앙 음성 버튼 → 녹음 UI
- 하단 탭: 채팅 / 갤러리

**채팅**
- 음성 메시지 버블(현재 목데이터), 텍스트 입력 전환

**갤러리**
- 업로드 날짜 + 업로더만 표시(최소 스펙)

**프로필 관리**
- 이름 수정, 사진 다시 찍기(추후 아바타 재생성 트리거)

---

## 5. 지금 만든 것 vs 아직 없는 것

이번 1차 구현은 **UI 전체를 목데이터로 완성**한 상태다. 백엔드 연동은 아직 없다.

**만든 것**
- 8개 화면 전체 라우팅, 컴포넌트, 상태 UI(녹음 중, 알림 토스트 등)
- 프론트 데이터 모델(`FamilyMember`, `ChatMessage`, `GalleryItem`)

**아직 없는 것 (백엔드 문서 대조 후 확인된 갭)**
- **보이스팩 등록 화면이 없다**: `familog-ai`의 `/voicepack`은 가입 시 "지정 문장 낭독"을 요구하는데, 지금 온보딩엔 사진 촬영(카메라 스캔)만 있고 음성 녹음 단계가 없다. 추가 화면이 필요하다.
- 아바타/낭독 상태값(PENDING/PROCESSING/READY/FAILED) 폴링 로직이 없다 — 지금은 정적 이모지 아바타, 정적 mock 메시지.
- 실제 REST 연동, 멤버 식별자/인증 흐름이 없다.

---

## 6. 발표 어필 포인트 (프론트 관점)

> 시니어는 읽지 않는다, 본다·듣는다 → 그래서 홈 화면은 텍스트가 아니라 큰 아바타 격자 → 탭 한 번이 전부인 인터랙션 → 로딩이 보이지 않게 상태 폴백을 화면에 미리 설계

백엔드의 "어떤 상태에서도 화면은 뭔가를 보여준다"(PENDING이어도 플레이스홀더) 원칙을 프론트가 이어받아, 아바타/오디오가 아직 준비되지 않았을 때도 화면이 멈추지 않도록 설계했다(플레이스홀더 이모지 아바타, 목데이터 피드).
